Now we need to play around with the CSS attributes of these two new elements to get them nicely spaced and positioned along the center line of the banner, and adjust the size of the font for the web site title. The easiest way to vertically center elements is to use the 'position' attribute with 'relative' and the 'top' attribute with a negative value to make it move up or a positive value to move it down. For both elements we use the 'display' attribute with 'inline-block' to get them to line up side by side.

.logo
{
    width:110px;
    display:inline-block;
    margin-left:1em;
    position: relative;
    top:7.7px;
}

.title
{
    display:inline-block;
    font-size:5em;
    background-color:aqua;
    margin-left:0.1em;
    position:relative;
    top:-22px;
}

By setting 'position' to 'relative', any value you then apply to the 'top' attribute moves the element up or down by that length rather than moving the top edge of the element literally to that position. A negative value means that the element will be moved up by that length while a positive value means that the element will be moved down by that length. This is what I mean when I say you just have to experiment with different combinations of CSS attributes and values until you get the elements to behave as you want them to.

So short of using the DWT method that leave the javascript method that WILL work with web pages on a desktop or laptop. So what we will do is create a new file called 'common.js'. And in this file we will code a javascript function that writes the HTML code for our navigation structure.

function CreateNavigationStructure()
{
    document.write("<nav id=\"nav\" class=\"nav\">");
    document.write("<a class=\"nav_link\" href=\"index.html\">Home</a>");
    document.write("<br/>");
    document.write("<a class=\"nav_link\" href=\"products.html\">Products</a>");
    document.write("<br/>");
    document.write("<a class=\"nav_link\" href=\"services.html\">Services</a>");
    document.write("<br/>");
    document.write("<a class=\"nav_link\" href=\"contact.html\">Contact</a>");
    document.write("</nav>");
}

So what we have done is copied the all the HTML tags of our navigation structure and pasted them all inside our javascript function' CreateNavigationStreucture()'. Then:

  • We have added document.write(" to the front of each line.
  • We have added "); to the end of each line.
  • We have 'escaped' each double quote character, in the middle of the string, by adding a back slash (\") so that those characters become part of the string rather than being interpretted as terminting a string or beginning a new string.
So this function automates the typing of the HTML code for the navigation structure. So next we need to include 'common.js' in 'index.html' via HTML - that you CAN do, and here is how:

Javascript login form

Forms contain a bunch of inputs with which the user can enter assorted data that is to be processed by the web page to produce an out come. Among the simplest examples is a login form where the user enters a password in a text field and then clicks a 'login' button. Web page code then checks that the password is valid and, if it is, then hidden page content is revealed to the user.

This is password checking is normally done within the secure environment of the web server, and with PHP code. However it is also possible to do it with Javascript, albeit not all that securely. The server aspects of HTML forms will be discussed briefly here, however the data processing of forms will be done with javascript. You start a form like this:

<form target="..." action="..." method="...">

</form>

Now if you can remember form the HTML part of the course those tage properties have the following purposes:

  • target: there are 4 valid values for this property:
    • _blank: this will cause the results for form submission and data processesing to be displayed in a new 'child' web browser tab.

    • self: this will cause the results for form submission and data processesing to be displayed in the same web browser tab as the form. This is the default if you do not specify the 'target' property.

    • parent_: for all intents and purposes this has the same effect as '_self'. You are unlikely to be coding a form in a way that this is not the case.

    • _top: for all intents and purposes this has the same effect as '_self'. You are unlikely to be coding a form in a way that this is not the case.
    You are unlikely to need to use '_top' and '_parent'.

  • action: this is usually a local file name. It can be the same .html file name as the one the form is contained in, or it can be a different .html file name. It can also be the URL of an external .html web page.

  • method: there are 2 valid values:
    • get: this results in the form data being visible, appended to the URL of the web page. It is good for debugging purposes, but not all that secure.

    • post: this results in the form data being hidden, and is the more secure method for data transfer to the web server.

    NOTE: this is only useful if the web page is being accessed from a web server and the form data can be accessed through one of the server languages, e.g. PHP. It is not relevent if you are processing the form data via javascript.

You could just code the details of all your products manually in HTML, and that might be just fine if you had only half a dozen products. But what if you had a few hundred products? Coding the details of hundreds of products individually would take you a VERY long time and is a very inefficent way of doing the coding. So instead let's code a javascript data structure to store the details of all your products so they can be added much more quickly and efficiently. Then we can code a javascript function to read the text details from it and generate the necessary HTML tags using that text.

So what sort of data structure do we need? Well what sort of details do we need to store for each product? They would include the following surely:

  • Product name / text
  • Price / decimal number
  • Product code / text or integer
  • Image name / text
  • Postage cost / decimal number

  • Paypal button / HTML text

So we are going to need a data structure that is capable of storing different data types, and clearly a javascript 'object' is clearly going to be the best choice. In C these are refered to as 'structs' or structures, in Pascal they are called 'records' and in Python they are referred to as 'collections'. In contrast arrays are used to store identical data types.

One object will store the details of one product, so how do we store a bunch of products? Well we could create an array of the above objects. All the elements of an array must be the same data type but there is no rule that says that arrays can only contain strings or integers etc. They can also contain identical objects.

However just keep in mind that the rules with javascript are some what more loose than they are in C. In C the elements in an array must of strictly identical data types, but in Javascript they can vary. But it is still a good idea to assume the C rules, otherwise your code can potentially get very complicated and hard to debug. Drawing

To create drawings you must first add a canvas element, with the 'id' property, to your page.

<canvas id="my_canvas"></canvas>

OR

<canvas id="my_canvas" />

Then you need to setup a drawing context for canvas as in the following code:

let canvasDrawing = document.getElementById("my_canvas"); let context2dCanvas = canvasDrawing.getContext("2d");

So here is a list of all the drawing functions you have available to you.

Drawing methods

  • fillRect(nLeft, nTop, nWidth, nHeight): Draws a "filled" rectangle.
  • strokeRect(nLeft, nTop, nWidth, nHeight): Draws a rectangle (with no fill).
  • clearRect(nLeft, nTop, nWidth, nHeight): Clears specified pixels within a rectangle.

Parameters

  • nLeft: the horizontal position of the top left corner of the rectangle.
  • nTop: the vertical position of the top left corner of the rectangle.
  • nWidth: the width of the rectangle.
  • nHeight: the height of the rectangle.

Examples

let canvas1 = document.getElementById("canvas1");
if (canvas1)
{
    let ctx1 = canvas1.getContext("2d");
    if (ctx1)
    {
        ctx1.fillRect(10, 40, 80, 80);
        ctx1.strokeRect(100, 40, 80, 80);
        ctx1.fillRect(190, 40, 80, 80);
        ctx1.clearRect(200, 50, 20, 20);
    }
}

Path methods

  • beginPath(): Begins a new path or resets the current path.
  • closePath(): Adds a line to the path from the current point to the start.
  • isPointInPath(): Returns true if the specified point is in the current path.
  • moveTo(nX, nY): Moves the path to a point in the canvas (without drawing).
  • lineTo(nX, nY): Adds a line to the the path.
  • fill(): Fills the current path.
  • rect(): Adds a rectangle to the path.
  • stroke(): Draws the current path.

Parameters

  • nX: the horizontal position of the point.
  • nY: the vertical position of the point.

Examples

let canvas2 = document.getElementById("canvas2");
if (canvas2)
{
    let ctx2 = canvas2.getContext("2d");
    if (ctx2)
    {
        ctx2.beginPath();
        ctx2.moveTo(10, 10);
        ctx2.lineTo(30, 20);
        ctx2.lineTo(50, 50);
        ctx2.lineTo(80, 120);
        ctx2.lineTo(30, 125);
        ctx2.lineTo(10, 10);
        ctx2.stroke();
        ctx2.closePath();
        ctx2.beginPath();
        ctx2.rect(90, 40, 60, 40);
        ctx2.stroke();
        ctx2.fillStyle = "red";
        ctx2.fill();
    }
}

Circles and curves

  • bezierCurveTo(nStartX, nStartY, nMidX, nMidY, nEndX, nEndY): Adds a cubic Bézier curve to the path.
  • arc(nCenterX, nCenterY, nRadius, nStartAngle, nEndAngle, bCounterClockwise): Adds an arc/curve (circle, or parts of a circle) to the path.
  • arcTo(nStartX, nStartY, nEndX, nEndY, nRadius): Adds an arc/curve between two tangents to the path.
  • quadraticCurveTo(nStartX, nStartY, nCtrlX, nCtrlY, nEndX, nEndY): Adds a quadratic Bézier curve to the path.

Parameters

  • nStartX: Start x coordinate of the curve.

  • nStartY: Start y coordinate of the curve.

  • nMidX: Mid x coordinate of the curve.

  • nMidY: Mid y coordinate of the curve.

  • nEndX: End x coordinate of the curve.

  • nEndY: End y coordinate of the curve.

  • nCenterX: x coordinate of the circle center.

  • nCenterY: y coordinate of the circle center.

  • nCtrlX: x coordinate of the quadratic curve control point.

  • nCtrlY: y coordinate of the quadratic curve control point.

  • nRadius: radius of the circle.

  • bCounterClockwise: Optional and the default is false. Draws the curve clockwise if false and anticlockwise if true.

Examples

let canvas3 = document.getElementById("canvas3");
if (canvas3)
{
    let ctx3 = canvas3.getContext("2d");
    if (ctx3)
    {
        ctx3.beginPath();
        ctx3.moveTo(5, 10);
        ctx3.bezierCurveTo(20, 100, 30, 150, 50, 40);
        ctx3.stroke();
        ctx3.beginPath();
        ctx3.arc(70, 75, 20, 0, 2 * Math.PI);
        ctx3.stroke();
        ctx3.beginPath();
        ctx3.arc(115, 75, 20, 0, 1.7 * Math.PI);
        ctx3.stroke();

        // Start a path
        ctx3.beginPath();
        ctx3.moveTo(150, 10);
        // Create a horizontal line
        ctx3.lineTo(180, 10);
        // Create an arc
        ctx3.arcTo(220, 10, 200, 80, 30);
        // Create a vertical line
        ctx3.lineTo(210, 100);
        // Draw the path
        ctx3.stroke();

        ctx3.beginPath();
        ctx3.moveTo(220, 20);
        ctx3.quadraticCurveTo(240, 200, 280, 20);
        ctx3.stroke();
    }
}

Text

  • direction: Sets or returns the direction used to draw text.

    Values: context.direction = "text-direction", either left to right / "ltr" or right to left / "rtl".
    Please refer to the CSS section for details about this CSS attribute.
    E.G. context.direction = "ltr";


  • fillText(strText, nStartX, nStartY, nMaxWidth): Draws "filled" text on the canvas.

  • font: Sets or returns the font properties for text content.

    Values: context.font = "font-style font-variant font-weight font-size font-face"
    Please refer to the CSS section for details about these CSS attributes.


  • measureText(strText): Returns an object that contains the width of the specified text.

    Parameter: strText, contains the text that is to be measured given the currently set font...measureText(strText).width and measureText(strText).height.


  • strokeText(strText, nStartX, nStartY, nMaxWidth): Draws text on the canvas.

  • textAlign: Sets or returns the alignment for text content.

    Values: context.textAlign = "text-align"
    Please refer to the CSS section for details about the CSS 'text-align' attribute.


  • textBaseline: Sets or returns the text baseline used when drawing text. Values: context.textBaseline = "text-baseline"
    Please refer to the CSS section for details about the CSS 'text-baseline' attribute.

Parameters

  • strText: the text to dsiplay.

  • nStartX: the x coordinate of the start position of the text.

  • nStartY: the y coordinate of the start position of the text.

  • nMaxWidth: the maximum width that the text can occupy. Optional and the default is the full width of the text.

Examples

Colors, styles and shadows

  • addColorStop(): Specifies the colors and stop positions in a gradient object.

  • createLinearGradient(): Creates a linear gradient (to use on canvas content).

  • createPattern(): Repeats a specified element in the specified direction.

  • createRadialGradient(): Creates a radial/circular gradient (to use on canvas content).

  • fillStyle: Sets or returns the color, gradient, or pattern used to fill the drawing.

  • lineCap: Sets or returns the style of the end caps for a line.

  • lineJoin: Sets or returns the type of corner created, when two lines meet.

  • lineWidth: Sets or returns the current line width.

  • miterLimit: Sets or returns the maximum miter length.

  • shadowBlur: Sets or returns the blur level for shadows.

  • shadowColor: Sets or returns the color to use for shadows.

  • shadowOffsetX: Sets or returns the horizontal distance of the shadow from the shape.

  • shadowOffsetY: Sets or returns the vertical distance of the shadow from the shape.

  • strokeStyle: Sets or returns the color, gradient, or pattern used for strokes.

Transformations

  • scale(): Scales the current drawing bigger or smaller.

  • rotate(): Rotates the current drawing.

  • translate(): Remaps the (0,0) position on the canvas.

  • transform(): Replaces the current transformation matrix for the drawing.

  • setTransform(): Resets the current transform to the identity matrix. Then runs transform().

Image drawing

  • drawImage(): Draws an image, canvas, or video onto the canvas.

Image information & pixel manipulation

  • createImageData(): Creates a new, blank ImageData object.
  • getImageData(): Returns an ImageData object that copies the pixel data for the specified rectangle on a canvas.

  • ImageData.data: Returns an object that contains image data of a specified ImageData object.

  • ImageData.height: Returns the height of an ImageData object.

  • ImageData.width: Returns the width of an ImageData object.

  • putImageData(): Puts the image data (from a specified ImageData object) back onto the canvas.

Compositing

  • globalAlpha: Sets or returns the current alpha or transparency value of the drawing.

  • globalCompositeOperation: Sets or returns how a new image are drawn onto an existing image.

Other methods

  • clip(): Clips a region of any shape and size from the original canvas.

  • save(): Saves the state of the current drawing context and all its atributes.

  • restore(): Restores the previously saved state and attributes.
