Web programming involves the creation of text based files containing codes and instructions, that are read by a web browser and used to render the visual elements that make up the web page. It involves 3 distinct types of codes or instructions:

  • HTML tags - acronym for Hypertext Markup Language
        Used to construct the visual and non-visual elements of the page.

  • CSS - acronym for Cascading Style Sheets
        Used to arrange and format the visual elements on the page, and to implement the simple parts of user interaction with the web page.

  • Javascript
        Used to implement the more complex parts of user interaction with the web page.

A web site consists of one or more HTML text files, with the extension .htm or .html, inter-linked together so that the user can switch from one page to another by clicking the hyperlinks. Rather than explicitly opening and closing files. A web site will contain one 'home' or default page. Depending on the web server the web site is hosted on, the home page is expected to be named any of 'index.htm', index.html', 'default.htm', 'default.html', 'index.php', 'default.php', 'index.asp', 'default.asp'. There may be other possibilties too. Your web hosting service will tell you what the HTML file for your home page should be named.

HTML files that use PHP or ASP on the web server will need to have the .php and .asp file extensions. If you don't give your HTML files those extensions then any PHP or ASP code inside them may be ignored by the web server. Other server languages may require different file extensions that flag to the web server what server language they are using.

Generally speaking each page in the web site has the same layout and color scheme, and with some form of navigation structure containing hyperlinks to all of the pages in the web site. But each page will have unique content in one part of the page. Here is a typical structure:

The header->content nearly always contains the name of the company, group or individual. The main->sidebar often contains a navigation structure with link to all the web pages in the web site. The main->content contains the unique content of each web page. The footer->content often contains a secodary navigiation structure and contact information. But there is wide variation among web sites and there is no right or wrong answer. It is what ever you are your clients find aesthetically pleasing.

What software should you use to create your web site and edit the pages? Well that is matter of personal preference. I prefer MS Expression Web to do the bulk of editing of my web sites. This software is about 15 years old and is deprecated, or no longer maintained by Microsoft. That makes it FREE and I have not managed to find any web editing software that matches its feature of:

  • A WYSIWYG page editing tab, that lets you edit your web pages in much the same way that you edit a Microsoft Word document. WYSIWYG = What You See Is What You Get


  • A split tab that you can use to edit your web page like a word document but view the HTML and CSS at the same time. This gives you a quick way to learn about all the HTML tags and CSS styling.


  • A code tab you directly edit the HTML, CSS and Javascript code.


  • A site creation wizard that will create an entire web site for you, complete with all the navigation links, based on a small number of themed templates that you can choose from.
  • Automatic handling of Dynamic Web Templates.

    These allow you to create a master web page, a .dwt file, that contains the standard layout and features that all other HTML web pages (that use the .dwt fie) will adhere to. When you change the .dwt file MS EXpression Web automatically updates all the other.html web pages. The .dwt file contains editable regions that you can freely edit in each .html web page. The remaining code on your .html files are highlighted with an orange background and is, strictly speaking, not editable.



  • Flagging of HTML syntax errors, such as missing < and > characters and unterminated strings etc.


  • Flagging of mismatched tags, such as a <p> without a matching </p>.

There is one thing that you should be aware of when using MS Expression Web, and that is it uses the HTML4 standard where as modern web browsers use the HTML5 standard. This means that there are HTML tags and CSS attributes that were valid in HTML4 that may no longer work in modern web browsers that use HTML5. And there may be HTML tags and CSS attributes that work in modern web browsers, but that won't work (and will be flagged as an error) in MS Expression web. However these are few in number and are unlikely to cause you much troube as a beginner. If you happen to use any CSS attributes or tags that MS Expression Web flags as errors, then just ignore the error and use them anyway.

You can also use Microsoft Visual Studio Code or VSCode. You can download that software from "here.

Try them all, and many other web editing software out there, and see which one you like the best.
