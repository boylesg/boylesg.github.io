Introduction

What is CSS and what can you do with it? The lists of things you can do with it is VERY long, but here are some things you can do with it:

  • Make elements visible or invisible.
  • Change the back ground color of elements.
  • Change the borders of elements:
    • Style
    • Width
    • Color
    • Thickness
    • Pointed or curved corners
  • Change fonts
    • Style
    • Color
    • Weight
    • Italic
    • Underlined
    • Strike through
  • Margins around elements
  • Padding within elements
  • Mouse hover characteristics (any of the above and below) of elements
  • Mouse pointer types
  • Height and width of elements
  • How overflow within an element is handled
  • Alignment of elements relative to other elements
  • Alignment of text within

There are just too many for the average beginner, or even seasoned programmers, to remember. So only a selection of the most frequently used properties will be detailed here. Refer to this web page (W3 Schools CSS Reference) for a full list of CSS properties that you can manipulate.

Applying CSS

There are 4 ways you can apply CSS attributes to elements in your web pages:

  • Via the 'style' property in HTML tags

  • Via the 'class' property in which you can apply 1 or more CSS classes, seperated by a space.

  • Via the 'id' property.

  • Via tag names.

The Style Property

The style property is applied to a HTML tag like this:
<p style="background-color:blue;color:red">XXXXX</p>

XXXXX

In this example to CSS attributes have been applied, 'background-color' and 'color'(the color of any text inside the element. Any invalid CSS attribute names are ignored. Following the attribute name there must be a colon :, and following that value for that CSS style. In this case the 'background-color' is set to blue and the (text)'color' attribute is set to red. Any invalid attribute values are also ignored.

This method is suitable for applying 3 or less CSS attributes, or combinations of more than 3 CSS attributes that are to be used once on the web page. For longer lists of CSS attributes, and combinations of attributes that are used repeatedly on different elements throughout the web page, then the class or id methods are more efficient.

The class property

CSS classes selectors are implemented like this:

You can apply multiple class selectors by seperating the names by a space like this: <p class="Border Text">HELLO WORLD</p>

HELLO WORLD

Or you can apply a single class selector like this: <p class="Border">HELLO WORLD</p>

HELLO WORLD

The advantage of this method are that:

  • You can apply one or more class selectors to each element, which is great if you want to seperate different CSS properties into logical collections, as in the above examples.

  • You can use the id properties of different elements independantly of each other.

The id property

CSS classes selectors are implemented like this:

You can apply id selectors like this: <p id="Paragraph">HELLO WORLD</p>

HELLO WORLD

This method has two distinct disadvantages:

  • You cannot apply more one group of CSS attributes like you can with the class method. You have to combine all the attributes into one selector and this makes your CSS code less flexible and re-usable for other elements in your web pages.

  • If you have multiple elements that need the same group of CSS attributes then they all need to have the same id. That makes it impossible to 'get' all of these elements via JavaScript document.getElementById("...").

Where can CSS classes be listed?

Between style tags <style>...</style>

These tags can be any where within your web page because this element is not a visible one. But typically your style section is located between the head tags at the top of your .html/.htm file, as in this example. You would typical include your CSS classes here if they are only used within the page and no where else. If you need to use them in many of your web pages then you should list them in your .css file

In style.css that is included in your web page

Your .css file does not necessarily have to be named 'style', however that is the file name that is typically used (or variations of it). This file can be included in all of your web pages and act as a central repositry for all your CSS code. You include your .css file via the <link> tage, like this: <link href="../styles/style2.css" rel="stylesheet" type="text/css" />. Background colors and images

background-color

This attribute changes the background color of an element from the default white to what ever color you specify. Click here to find out the ways you can specify a color.

E.G.

  • background-color: red;
  • background-color: rgb(100, 20, 250);
  • background-color: rgba(100, 20, 250, 0.5);
  • background-color: hsl(50, 50%, 80%);
  • backgroud-color: hsla(50, 50%, 80%, 1.0);

background-image

You specify a background image using the JavaScript function url("..."). You can specify a local image file, e.g. url("images/Image.jpg"), or you can specify and external image file, e.g. url("https://cdn.pixabay.com/photo/2016/11/29/05/45/astronomy-1867616_1280.jpg"). The default mode is to tile the image if it is to small to cover the designated area of the screen.

background-repeat

This attribute controls how the image covers the background. Allow values are:

  • repeat: this causes the background image to be tiled across the space in both the horizontal and vertical directions. The image will be clipped aling the right and bottom edges if necessary.

  • repeat-x: this causes the background image to be tiled horizontally only.

  • repeat-y: this causes the background image to be tiled vertically only.

  • no-repeat: this cause the image to be centered in the area and not tiled. If it is too small to cover the area then there will be blank margins around it.

  • space: this prevents clipping o the image when it is tiled across the area. There may be blank spaces along the right and bottom edges if image cannot fit in the space available.

  • round: this causes the image to be stretched horizontally and/or vertically so that it can be tiled across the area evenly wit no clipping of the image at the left and bottom edges.

background-position

This attribute controls where the position in the area where tiling with the image starts:

  • left top: this means that the tiling begins at the top left corner of the area.

  • left center: this means that the tiling begins at the center of the left edge of the area.

  • left bottom: this means that the tiling begins at the bottom left corner of the area.

  • right top: this means that the tiling begins at the top right corner of the area.

  • right center: this means that the tiling begins at the middle of the right edge of the area.

  • right bottom: this means that the tiling begins at the bottom right corner of the area.

  • center top: this means that tiling starts at the center of the top edge of the area.

  • center center: this means that tiling starts at the center of the area.

  • center bottom: this means that tiling starts at the center of the bottom edge.

  • X% Y%: this means, that if the area has width of 300 pixels and height of 200 pixels, then tiling starts at X% of 300 pixels in the horizontal direction and Y% of 200 pixels in the vertical direction.

  • Xpx Ypx: this means that tiling begins at pixel number X and pixel number Y in the area.

background-position-x

This attribute controls where the position in the area where tiling with the image starts:

  • left: this means that the tiling begins at the left side of the area.

  • center: this means that the tiling begins at the horizontal center of the area.

  • right: this means that the tiling begins at the right edge of the area.

  • X%: this means, that if the area has width of 300 pixels and height of 200 pixels, then tiling starts at X% of 300 pixels in the horizontal direction.

  • Xpx: this means that tiling begins at pixel number X in the horizontal direction of the area.

background-position-y

This attribute controls where the position in the area where tiling with the image starts:

  • top: this means that the tiling begins at the top edge of the area.

  • center: this means that the tiling begins at the vertical center of the area.

  • bottom: this means that the tiling begins at the bottom edge of the area.

  • Y%: this means, that if the area has height of 200 pixels, then tiling starts at Y% of 200 pixels in the vertical direction.

  • Ypx: this means that tiling begins at pixel number Y in the vertical direction of the area.

background-size

With this attribute you can change the size of the image or its widht and height independantly. Allowed values are:

  • auto: default value - the background image is displayed in its original size.

  • length: expects one or two values - the first value specifies the width of the image and the second value is the height of the image. Values are expressed in pixels like this 50px.

  • percentage: also expects one or two values - the width and height of the image. Except the values are expressed as percentages of the width and height of the area, like this 50%.

  • cover: this causes the image to stretch or shrink to cover the entire area. the image may be clipped vertically or horizontally.

  • contain: this causes the image to stretch or shrink to fill the entire width or height of the area without clipping.

background-origin

You can use this attribute to set the starting point of a background image. Allowed Values are:

  • padding-box: default value - the background image starts from the upper left corner of the padding edge. If the padding of the element is 0 pixels then the background image starts at the border edge.

  • border-box: the background image starts from the upper left corner of the border edge and it will be clipped if a border style other than 'none' is set.

  • content-box: the background image starts from the upper left corner of the content edge.
If you have more than one background image then you can set them with different values by seperating the values with spaces, e.g. in the case of 2 background images: background-origin: padding-box content-box, the first image gets the fist value and the second image gets the second value etc.

Display & Visibility

display

This attribute controls how and element is displayed. Allowed values are as follows:

  • inline: this is the default display mode for <span> and elements derived from it e.g. <a>, <img>, <b>, <i> and <u> etc. Inline elements take on the minimum necessary width and height in order to display their contents. If they don't have any contents then they will not be visible. They ignore 'width' and 'height' attributes.

    <span style="background-color:darkblue">XXXXXXXXXX</span><span style="background-color:darkgreen">YYYYYYYYYY</span>
    XXXXXXXXXXYYYYYYYYYY

  • block: this is the default display mode for <div> and elements derived from it e.g. <h1...h6>, <p>, <form>, <ol> and <ul> etc. Block elements can be displayed without any contents if you specify the 'width' and 'height' attributes, otherwise they will not be visible.

    <div style="background-color:darkblue"<XXXXXXXXXXYYYYYYYYYY</div>
    XXXXXXXXXX
    YYYYYYYYYY

    <div style="background-color:darkblue;height:3em;width:10em;"></div><div style="background-color:darkgreen;height:3em;width:10em;"></div>

  • contents: this display mode causes the element to display all the child elements it contains, but not itself. Although the parent element will take up the same space as it would if it were visible.


  • inline-block: this causes the element to display like a block except that successive inline-block elements can be displayed side by side like <span>.
There are many others but I will not cover them here because you are unlikely to use them unless you are an advanced web programmer.

visibility

This attribute controls the visibility of an element. The following values can be used with it:

  • visible: this causes the element to be displayed on the web page as normal.

  • hidden: this causes the element to be hidden, however it still occupies the same area on the page.

  • collapse: this value is for tables only and causes the row or column to be invisible and not take up any space.

Positioning

position

This attribute controls where and how and element is positioned. Recognozed values are as follows:

  • static: this is the default value and it means that elements will be positioned in the same order as they appear in the code.
  • absolute: this causes the element to be positioned relative to its parent element.
  • fixed: this cause the element to be positioned relative to the browser window.
  • relative: this causes the element to be positioned relative to its static position, and it means you can, for example, give it a negative 'top' attribute in order to move it up a little.
  • sticky: this cause the element to bbe positioned relative to the parent lements scroll position.
