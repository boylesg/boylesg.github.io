Introduction

What is JavaScript

JavaScript is a programming language that allows you to implement more complex features on web page. There are 3 layers to any web page:

  • HTML: allows you to code the basic structure of the web page.
  • CSS: allows you to modify the formatting of the basic structures within the web page, i.e. change back ground colors, text colors, borders and fonts etc. as well as implement simple intereactactivity such as mouse pointer hover effects etc.
  • JavaScript: allows you to implement far more complex types of intereactivity such as displaying timely content updates, interactive maps, animated 2D/3D graphics, scrolling video jukeboxes, etc.

It is important to know that JavaScript and Java are two unrelated programming languages that are used on different platforms and for different types of software. JavaScript is an 'untyped' language, which means you don't need to specify a data type for any variables you declare. And you can assign data of any type to your variable. In short you can do this in JavaScript:

let MyVariable = 0;
MyVariable = "Hellow world!";
MyVariable = 3.45;

In contrast, if you tried this in C (a strictly typed language), then you would get syntax errors because, for example, you can't assign the string value "Hello world!" to a variable that has been delcared as an integer variable.

JavaScript is also an interpretted language, meaning that your text based code is converted to binary CPU instructions one line at a time by the web browsers, and executed. This is in contrast to a copiled language like C where all your text based code is converted to binary CPU instructions before any of the instructions are executed by the CPU. Because it is an interpretted language it will work on any device and under any operating system because the specific web browser deals with the specifics of the video hardware of the device - your JavaScript code does not have to.

The syntax of JavaScript is identical to C, so of you are a C programmer then you will have little trouble getting the hang of JavaScript. It is also strictly a client side or web browser language - it gives you no access to the web server functions. The need for security also means there are limits to what you can do with it in the web browser. You cannot do any the following with JavaScript:

  • File handling.
  • Run programs on the user's system.
  • Access the user's contacts or personal data.
  • Access the user's OS settings.
Your code is strictly confined to the web browser's internal environment.

How do you include JavaScript in your web page?

You include JavaScript in your web page by using the <script> tag like this:

<script type="text/javascript">
........
</script>

You can have as many of these script tags through out your web page as you want or need. And it does not matter where in your web page you include script tags - they are remain invisible to anyone viewing the web page. Any variables or functions declared within script tags at the top if the web page will be available for use within script tags further down in your web page.

Syntax and run time errors in JavaScript

If your JavaScript contains a syntax error or a run time error then the we browser will simply stop executing your code from that point on. So it can lead to confusing faults in your web page. Elements after the JavaScript error may not display properly or at all and elements that rely on your JavaScript code after the error will just fail to be interactive.

The best way to detect such error is to use the 'developer tools' in either:

  • Firefox Developer Edition





  • Google Chome





Any syntax or run time errors will be flagged in the 'console' tab, and the error message will include the file name and the line number of the error.

Special characters

These are also known as 'escaped' characters and provide a way of including non-printing characters and certain reserved characters in a string:

  • Single quote: \', e.g. "\'"
  • Double quote: \", e.g. "\""
  • Ampersand: \&, e.g. "\&"
  • Newline: \n, e.g. ""
  • Tab: \t, e.g. "\t"
  • Backslash: \\, e.g. "\\"
  • Carries return: \r, e.g. "\t"
  • Backspace: \b, e.g. "\b"
  • Form feed: \f, e.g. "\f"
  • Vertical tab: \v, e.g. "\v"

Essentially, as long as you have the reserved word of special charatcers enclosed in single or double quotes then it is allowed.

Variables

Naming rules

  • The first character must be a letter, dollar or underscore.
  • Contain any number and combination of letters, dollars, underscores and digits.
  • Letters: A to Z or a to z
  • Underscore: _
  • Dollar: $
  • Digits: 0 to 9
Variable names are case sensitive. So, for example msg, MSG, Msg, mSg are all considered to be unique variables. Also you cannot declare variables with the same spelling in the same scope, e.g. you cannot do the following inside a function:

function MyFunction()
{
    var msg;
    var msg;
}

The above is a variable naming conflict that occurs when you try to give two variables the same name. The following is however is valid:

function MyFunction()
{
    var msg;
    var Msg;
}

Remember that vaiable names are case sensitive.

Declaring a variable

Variables must be explicitly declared before they can be used. You can do this by using either the varor the let key words. The difference between these two key words are as follows:

  • var: e.g. var msg;

    • Variables declared with var have global scope, which means you can 'access' that variable any where in your JavaScript code on a given HTML web page. If you declare the variable inside a function then it is accessible from code that is not part of that function or from code that is part of another function. For this reason it is not recommended that you declare variables using the var key word for these reasons:

    • It can lead to variable naming conflicts if you want to use the same variable name in the same scope, e.g.

      function MyFunction1()
      {
          var msg;
      }

      function MyFunction2()
      {
          var msg;
      }

      Because var has been used to declare both variables they are in the same global scope and therefore they cannot have the same name.
    • It can lead to coding confusion, and hard to find runtime errors, where you get unexpected values in variables that, for example, are declared a different function and contain a value that is unrelated to the function your are coding, e.g.

      function GetUserName()
      {
          var User = "Fred Smith"; }

      function GetUserDOB()
      {
          // This would return "Fred Smith" rather than a valid date.
          return User;
      }

  • let: e.g. let msg;
    Variables declared with let are local, which means that, for example, a variable declared inside a function can ONLY be accessed by following code that is part of the function. You will not be able to access it from code that is not part of your function. E.G. There is no variable naming conflict in the following example.

    function MyFunction1()
    {
        let msg; }

    function MyFunction2()
    {
        let msg; }

    Because let has been used to declare both variables they are in different local scopes and therefore they can have the same name.

    Therefore, if you did the following, then it would cause a syntax error.

    function GetUserName()
    {
        let User = "Fred Smith"; }

    function GetUserDOB()
    {
        // This would cause a syntax error because the variable User has not been declared before use.
        // As far as this function is concerned the variable User, declared in the function GetUserName(), is out of scope and does not exist.
        return User;
    }

    Think of function MyFunction1 and function MyFunction2 as two remote Pacific islands, each with their own copy of the variable named msg and each copy containing a different value.

    It is strongly recommended that you get into the habit of declaring your JavaScript variables using the let keyword.

Simple data types

Simple data types can be divided into the following broad categories:

  • Integers or whole numbers: they can be either negative or positive, e.g. 1, 100, -99 and 0

  • Real or floating point or decimal numbers: they can be either psoitive or neagtive and consist of a whole number part and a fraction part, seperated by a decimal point, e.g. 0.234, 100.34, -99.478 and 0.0.

  • Strings or text: string of ASCII characters that form human readable text.

  • Boolean: this data type has only two possible values, either 'true' or 'false'.

JavaScript is not a 'typed' language. This means you don't have to specify, at declaration, what type of data your variable will store. And you can therefore assign ANY value to your variable at any time:

let MyVar = 0;
MyVar = "Hello world!";
MyVar = false;

This is in contrast to C where you must specify a data type for your variable, and there after you can only assign values to that variable of the correct data type:

int MyVar = 0;
MyVar = 100;

But if you tried the following in C then you would get a compile error:

MyVar = "Hello world!";

Objects

In C this type of data structure is referred to as a 'struct', in Pascal it is referred to as a 'record' and in Python it is referred to as a 'collection'. Essentially they are a number of variables, containing different data types, all stored together is a single container. Think of a manilla folder containing all the information pertaining to a student or a patient - a photo, name, address, age, email address, phone numbers,...

This is how you declare an object in JavaScript:

let Student = {
                             strSurname: "Smith",
                             strGivenNames: "Fred",
                             nAge: 56
                      }

You can access one of the fields like this:

    Student.strSurname

    Student["strSurname"]

You can assign a field a new value like this:

    Student.strSurname = "Jones";

    Student["nAge"] = 57;

Compound data structures

A compound data structure combines multiple simple data structures into a more complex ones. An example is an array of objects:

let arrayStudents = [
                                     {strSurname: "Smith", strGivenNames: "Fred", nAge: 56},
                                     {strSurname: "Jones", strGivenNames: "Tom", nAge: 61},
                                     {strSurname: "Fredricks", strGivenNames: "John", nAge: 77}
                               ];

arrayStudents[0].strSurname contains the value "Smith"
arrayStudents[1]["nAge"] contains the value 61
arrayStudents[2].strGivenNamescontains the value "John"
