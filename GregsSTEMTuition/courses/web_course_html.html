Introduction

HTML is based on tags that enclose the text to which the tags apply. An example is <h1>Title< /h1>, which refers to first or top level heading that causes the text 'Title' to be rendered in a large font.

  • <h1>: start of  first level heading
  • Title: the text to which the h1 tags apply.
  • </h1> end of first level heading

Refer to this web page (W3 Schools HTML Reference) for a full list of HTML tags that you can use in your web pages. However only a selection of HTML tags, that you are likely to use the most often, will be detailed here.

Tags have additional properties that can be applied. Some are specific to the tag while others are general and can be applied equally to any tag. They look like the following:

<p class="paragragh" style="color:blue;">This is a paragraph of text. </p>

In this example the <p> tag has two general properties applied to it: 'class' and 'style'. The actual contents of the property must be enclosed in double quotes.

But again here we will only detail those properties that are required and the ones you are most likely to use regularly. For other properties please refer to the above comprehensive list of HTML tags at W3 Schools.

There are two specific properties that you ought to use and they are:

For some Asian languages it may be "rtl" or right to left. You can also set it to "auto" in which case the web browser will decide the directionality of the text.
So the tag would look like this: <html xmlns=" http://www.w3.org/1999/xhtml" dir="ltr">....</html>

Hyperlinks

There is only one specific property for a hyperlink:

Divisions & Spans

<div>...</div>
<span>...</span>
Divisions and spans are generic structural elements that are used to build web page structures and layouts. They have no specific properties associated with them, but you can use a wide variety of general properties with them.

Divs are 'block' elements that display in the same way as paragraphs do. If you have 2 consecutive divs then the first one will display above the second with a line break between them. And you can set a width and height for them independantly of the contents of the division. Consider the example: <div>...</div> <div>...</div>

Spans are 'inline' elements that display in the same way as hyperlinks and images do. If you have 2 consecutive spans then they will display side by side with no line break between them. The width and height of the span is determined by its contents. Consider the example: <span>...</span> <span>...</span>

Escape Characters

Let's use the example of the < character. If you literally type the character SHIFT + ,< key in your text editor, as part of a paragraph of plain text, then the web browser is going to read that character and assume you are trying to specify a tag. It will read the next word from the plain text and then try and figure out what tag your are referring to. Chances are the next word will not specify a valid HTML tag name and your web browser will spit the dummy and probalby not dispaly the rest of your web page contents properly.

So how do you type are < character in HTML code? You do it like this: &lt;. The code 'lt' is an acronym for 'less than'. Logically the opposite character, >, is specified by typing &gt; into your HTML code. The code 'gt' is an acronym for 'greater than'.

So here is a list of HTML escape characters you are most likely to want to use:

ESCAPE SEQUENCE RENDERED NAME
&amp; & ampersand
&lt; < less than
&gt; > greater than
&quot; " quotation mark
&apos; ' apostrophe
&nbsp;   non-breaking space

There are many other more obscure characters, but I won't cover them here as you are highly unlikely to have any cause to use them. Just remember these ones.

</form>

As in a construct in which the user enters details like their name, email adddress and phone number etc. and then presses a button to do something with that data.

There are 6 specific properties that are worth mentioning:

  • action: This is either a URL of an external web page or a local HTML web page (it can be the same HTML web page that the form is on), to which the form data will be sent when the form is submitted. This property is required if you don't want the form data to disappear into thin air when the form is submitted


  • method: This specifies the way the form data is sent to the web server. Allowed values are as follows:
    • post: This essentially means that the form data is sent to the web server as text in a way that hides it from the user of the form.
    • get: This essentially means that the form data is sent to the web server as text appended to the URL of the web page in the web browser's search box.
    If this property is not specified then the default for the form is 'post'.


  • autocomplete: Allowed values are as follows:
    • on: This mean that typed text prediction is enabled.
    • off: This mean that typed text prediction is disabled.


  • name:
    • any alphanumeric text string specifying the name of the form. This name is sent along with the data from the other form inputs elements. It is handy if you have more than one form on the page and you can use to figure out which of your forms the user submitted.

There are a few other properties available but you are not likely to need them unless you are an advanced programmer.

How is form data sent to the web server?

  1. The data entered by the user into each input element is sent via the 'name' property. If the input does not have its 'name property' specified then any data the user entered into it is ignored by the web browser, and not sent to the web server.

  2. The form must have a input with the 'type' property set to 'submit'. The 'submit' input is simply a special type of button that automatically 'submits' the form when the user clicks it. It can have a name property or not. If you have multiple forms on a page then you can give each submit button a unique name and alternatively use it to figure out which form the user submitted.

If the submission 'method' of the form is set to 'get' then then the form data will appear as part of the web pages URL. Consider this simple login form:
        <form action="MyWebPage.html" method="get">
                <label for="password">Password&nbsp;</label>
                <input type="password" id="password" name="password" value=""/>&nbsp;
                <input type="submit" value="LOG IN" />
        </form>

 

If you enter 'XXXXXXXXXX' as the password and hit the submit button then this is the sort of thing you will see in the URL: MyWebPage.html?password=XXXXXXXXXX. So we have the URL with the name of the html file, then a ? symbol, then the name of the first input field (in this case 'password'), then the = symbol and the data entered by the user (in this case XXXXXXXXXX without quotes).

If there are additional inputs with a 'name' property set then you will see a & symbol next, then the contents of the 'name' of the next input field, then another = symbol and then the value of that inout field that was entered by the user. And so on till the last input field with its 'name' property set. E.G. MyWebPage.html?password=XXXXXXXXXX&Surname=Smith&GivenNames=Fred

If the submission 'method' of the form is set to 'post' then the form data is sent in the same manner except that it is hidden from the user and any one watching them enter data into your form.

POST Method

ADVANTAGE: More secure in that no one can see what password the user has entered for example.
DISADVANTAGE: It is harder to debug because you cannot readily see the form data.

GET Method

ADVANTAGE: It is easier to debug because you can readily see the form data that was entered.
DISADVANTAGE: A password entered by the user can potentially be seen by some one watching them..

How do you access the form data at the web server?

ASSUMPTIONS:

  1. You have specified the same HTML web page file that contains the form that sends the data.
  2. PHP is the server language installed on your web server.

Very briefly, some where in your HTML web page you would have the following code:
<?php
        $_GET["passord"];
        $_POST["passord"];
?>
One of these two expression would give you the string "XXXXXXXXXX" given the above example, which was the password entered by the user. Which one you use depends on what you set the 'method' property to in your form, method="get" or method="post".

Now you could compare that to a master password stored securely in a SQL database on the web server. If they match you cvould manipulate the HTML on the web page, before it is re-sent to the user's web browser, in order to make the rest of the content visible to the user.

It is important to understand that the web server will remove all the code between the tags <?php...?>, and the tags themselves, before the html file is downloaded again by the web browser. This will also cause the data entered form to be erased, as the HTML web page is rendered again from the HTML code by the web browser.
