Introduction

HTML is based on tags that enclose the text to which the tags apply. An example is <h1>Title< /h1>, which refers to first or top level heading that causes the text 'Title' to be rendered in a large font.

  • <h1>: start of  first level heading
  • Title: the text to which the h1 tags apply.
  • </h1> end of first level heading

Refer to this web page (W3 Schools HTML Reference) for a full list of HTML tags that you can use in your web pages. However only a selection of HTML tags, that you are likely to use the most often, will be detailed here.

Tags have additional properties that can be applied. Some are specific to the tag while others are general and can be applied equally to any tag. They look like the following:

<p class="paragragh" style="color:blue;">This is a paragraph of text. </p>

In this example the <p> tag has two general properties applied to it: 'class' and 'style'. The actual contents of the property must be enclosed in double quotes.

But again here we will only detail those properties that are required and the ones you are most likely to use regularly. For other properties please refer to the above comprehensive list of HTML tags at W3 Schools.

 

Headings

<h1></h1>
<h2></h2>
<h3></h3>
<h4></h4>
<h5></h5>
<h6></h6>

These work in the same way as the headings in MS Word and other word proessing software. Each has a default font size and color, that you can change via CSS if you wish. There are no specific properties that apply to these tags however you can apply the CSS properties 'class' and 'style' to them individually if you wish.

 

HTML

<html>
.
.
.
</html>

These tags are required, and all other tags and text on the page need to be between these. They tell the web browser where the HTML code starts and ends.

There are two specific properties that you ought to use and they are:

  • xmlns: this specifies a xml namespace and should be set to "http://www.w3.org/1999/xhtml". You can think of it like a code library that contains all the HTML tags available for you to use in your web page.

  • dir: this is the direction of the text and is there to support different languages that use different text directions. For English it should be set to "ltr" or left to right. For some Asian languages it may be "rtl" or right to left. You can also set it to "auto" in which case the web browser will decide the directionality of the text.
So the tag would look like this: <html xmlns=" http://www.w3.org/1999/xhtml" dir="ltr">....</html>

Title

This is a non-visual element that specifies the title of the page. Rather than being displayed within the browser window it is actually displayed on the browser page tab.

 

Body

These tags are also required and, by convention, all visual elements of your page should be between these tags. There are no specific properties for this tag and general properties are rarely applied.

Web pages do not always have these tags but you can use them if you wish. You might use them to list other non-visual page elements or use them to create a distinct region on your page for displaying particular common elements. There are no specific properties for this tag.

The reason for this is that multiple consecutive space characters, generated by hitting the space bar, are ignored by the web browser. They are for the purpose of formatting the HTML code in a text editor. In order to get the web browser to render more than one space character you must use & symbol followed by a text code designating the non-alphanumeric and non-punctuation character you want. In HTML coding these are referred to as escape characters. In the case of a space character the text code is 'nbsp', which is an acronym for 'non-breaking space'. The term 'breaking' refers to creating a new line of text.

There is also a special tag for creating a new line of text within a single paragraph and that is <br/>. This will result in any following text to appear on a new line but without any gap between the two lines of text, as would be the case if you enclosed both passages of text within separate sets of p tags.

 

Images

<image src="..." alt="..."></image>

or

<image src="..." alt="..." />

The /> is short hand for ></image>

There are two specific properties that you ought to use and they are:

  • src: this is required and specifies the name, and optional path, of the image file that is to be rendered in the web page.
    • If you don't specify a valid image file name, and the alt property is not specified or contains and empty string, then you will see a blank space in the web page.

  • alt: this specifies the text to display in place of the image of there is a problem rendering the image.

A typical image tag looks like this: <image src="images/MyImage.jpg" alt="images/MyImage.jpg" />. In this case the image file is found in a sub-folder named 'images'. Images are displayed 'inline' by default. To understand what that means then consider this example:
