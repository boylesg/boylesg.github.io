Introduction

What is CSS and what can you do with it? The lists of things you can do with it is VERY long, but here are some things you can do with it:

  • Make elements visible or invisible.
  • Change the back ground color of elements.
  • Change the borders of elements:
    • Style
    • Width
    • Color
    • Thickness
    • Pointed or curved corners
  • Change fonts
    • Style
    • Color
    • Weight
    • Italic
    • Underlined
    • Strike through
  • Margins around elements
  • Padding within elements
  • Mouse hover characteristics (any of the above and below) of elements
  • Mouse pointer types
  • Height and width of elements
  • How overflow within an element is handled
  • Alignment of elements relative to other elements
  • Alignment of text within

There are just too many for the average beginner, or even seasoned programmers, to remember. So only a selection of the most frequently used properties will be detailed here. Refer to this web page (W3 Schools CSS Reference) for a full list of CSS properties that you can manipulate.

Applying CSS

There are 4 ways you can apply CSS attributes to elements in your web pages:

  • Via the 'style' property in HTML tags

  • Via the 'class' property in which you can apply 1 or more CSS classes, seperated by a space.

  • Via the 'id' property.

  • Via tag names.

The Style Property

The style property is applied to a HTML tag like this:
<p style="background-color:blue;color:red">XXXXX</p>

XXXXX

In this example to CSS attributes have been applied, 'background-color' and 'color'(the color of any text inside the element. Any invalid CSS attribute names are ignored. Following the attribute name there must be a colon :, and following that value for that CSS style. In this case the 'background-color' is set to blue and the (text)'color' attribute is set to red. Any invalid attribute values are also ignored.

This method is suitable for applying 3 or less CSS attributes, or combinations of more than 3 CSS attributes that are to be used once on the web page. For longer lists of CSS attributes, and combinations of attributes that are used repeatedly on different elements throughout the web page, then the class or id methods are more efficient.

The class property

CSS classes selectors are implemented like this:

You can apply multiple class selectors by seperating the names by a space like this: <p class="Border Text">HELLO WORLD</p>

HELLO WORLD

Or you can apply a single class selector like this: <p class="Border">HELLO WORLD</p>

HELLO WORLD

The advantage of this method are that:

  • You can apply one or more class selectors to each element, which is great if you want to seperate different CSS properties into logical collections, as in the above examples.

  • You can use the id properties of different elements independantly of each other.

The id property

CSS classes selectors are implemented like this:

You can apply id selectors like this: <p id="Paragraph">HELLO WORLD</p>

HELLO WORLD

This method has two distinct disadvantages:

  • You cannot apply more one group of CSS attributes like you can with the class method. You have to combine all the attributes into one selector and this makes your CSS code less flexible and re-usable for other elements in your web pages.

  • If you have multiple elements that need the same group of CSS attributes then they all need to have the same id. That makes it impossible to 'get' all of these elements via JavaScript document.getElementById("...").

Where can CSS classes be listed?

Between style tags <style>...</style>

These tags can be any where within your web page because this element is not a visible one. But typically your style section is located between the head tags at the top of your .html/.htm file, as in this example. You would typical include your CSS classes here if they are only used within the page and no where else. If you need to use them in many of your web pages then you should list them in your .css file

In style.css that is included in your web page

Your .css file does not necessarily have to be named 'style', however that is the file name that is typically used (or variations of it). This file can be included in all of your web pages and act as a central repositry for all your CSS code. You include your .css file via the <link> tage, like this: <link href="../styles/style2.css" rel="stylesheet" type="text/css" />. Typically this would also be located between the <head> tags at the top of your .html/.htm file.
