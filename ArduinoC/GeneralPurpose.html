Arduino C Libraries - General Purpose

CArray

Purpose

  • To implement the functions of an array without having to worry about array index over run errors.

  • It is a template class where you specify the data type it will contain and the maximum size.

  • You use it like this: CArray<int, 100> arrayIntegers;

  • The class does not use dynamic memory (new, alloc and malloc), which can cause memory fragmentation with the limited memory available on Arduino microcontrollers.

  • The second template parameter (100 in the above example) specifies the size of a regular C array inside the class.

Dependancies: None

Queue functions:

        These functions allow you to treat the array as a queue or FIFO data structure.

        bool enqueue(DataType DataVal)

                 Add a new item at the end or array. The array length will increase by 1.
                 DataVal: new value to add.
                 Returns: true if the number of items in the array is less than the maximum length, otherwise false.

        bool dequeue(DataType &DataVal)

                 Remove an item from the beginning of the array.
                 All other items in the array will be moved toward the beginning of the array by one place.
                 The array length will decrease by 1  .
                 DataVal: The value removed is returned in this parameter.
                 Returns: true if the array is not already empty, otherwise false.

Stack functions:

        These functions allow you to treat the array as a stack or LIFO data structure.

        bool push(DataType DataVal)

                 Add a new item at the end of the array.
                 The array length will increase by 1.
                 DataVal: The value to be pushed.
                 Returns: true if the array is not already at its maximum length, otherwise false.

        bool pop(DataType &DataVal)

                 Remove an item from the end of the array.
                 The array length will decrease by 1.
                 DataVal: The value removed is returned in this parameter.
                 Returns: true if the array is not already empty, otherwise false.

List functions:

         These functions allow you to treat the array as a list data structure.

        bool add(DataType DataVal)

                 Add a new value at the end of the array.
                 The array length will increase by 1.
                 DataVal: The value to be added..
                 Returns: true if the array is not already at its maximum length, otherwise false.

        bool insertAt(const uint8_t nI, DataType DataVal)

                 Insert a new value into the array after the specified index.
                 The item at that index and all other after it will move towards the end of the array by one place.
                 The array length will increase by 1.
                 nI: The index the new value will be inserted at.
                 DataVal: The new value to insert.
                 Returns: true if the array is not already at its maximum length, otherwise false.

        bool removeAt(const uint8_t nI, DataType &DataVal)

                 Remove an item from the array at the specified index.
                 The value is returned in parameter DataVal
                 All other items after that index will move towards the front of the array by one place.
                 The array length will deccrease by 1.
                 nI: The index of the value that will be removed.
                 DataVal: The removed value is returned in this parameter.
                 Returns: true if the array is not already empty, otherwise false.

Array functions:

        These functions are the regular array indexing functions.

        bool setAt(const uint8_t nI, DataType DataVal)

                Change the value of the item at the specified indexto the new specified value.
                nI: The index of the value that will be changed.
                DataVal: The new value.
                Returns: true if the nI is in range, otherwise false.

        bool getAt(const uint8_t nI, DataType &DataVal)

                nI: The index of the value that will be fetched.

                Returns: true if the nI is in range, otherwise false.

        DataType operator [](const uint8_t nI)

                Return the value at the index specified by parameter nI

General functions:

CBufF, CFixedBuff, CDynamicBuff

Purpose

       To implement the functionality of an array buffer. Unlike a native C array, CBuff, CFixedBuff and CDynamicBuff intialise their
       internal arrays to 0's on declaration.

       CBuff  is a template class where you declare its maximum size like this: CBuff<100> buff. It avoids the use of
       dynamic memory which can cause memory fragmentation in the limited memory of an Arduino MCU.

       CDymanicBuff is a regula class where you declare the maximum size like this: CDynamicBuff buff(100). It does use dynamic
       memory and should be used sparingly and with caution on Arduino MCUs.

       CFixedBuff allows you to use an existing external native byte array as the internal array for the class. You use it like
       this:

       char strText[100];
       uint8_t arrayInts[100];
       CFixedBuff buff1(arrayInts, 100, true), buff2(arrayInts, 100, false);

       All 3 classes are dervived from CBuffBase and objects can be used interchangeably as pointers:

       CBuff<100> buff1;
       CDynamicBuff buff2(100);
       CBuffBase pBuff = &bff1;
       pBuff = &buff2;

Dependancies: None

Functions:

               strBuff:a pointer to a byte array. It can be a char, uint8_t or int8_t array.
               nBuffSize:the capacity of the array.
               bClear: true if you want the array to be initialised to 0's, false if not.

        CDynamicBuff(const uint16_t nBuffSize)

                nBuffSize: the maximum capacity of the buffer.

        void empty()

                 Gets the value at the specified index.

                 nI: the index of the value to be fetched.

        operator char*()

                 Converts the buffer object to a native C string.

                 Allows you to treat the buffer as a queue or FIFO data structure.
                It adds the specified new value to the end of the internal array.

                 Returns: true if the length of the buffer is less than it maximum capcity, otherwise false.

                 Allows you to treat the buffer as a queue or FIFO data structure.

                 Returns: the value removed if the array is not empty, otherwise 0.

                 Allows you to treat the buffer as a stack or LIFO data structure.

                 cCh: the new value to be added.

                 Allows you to treat the buffer as a stack or LIFO data structure.
                It removes the value at the end of the internal array and returns it.

                 Returns: the value that was removed if the inernal array is not empty, otherwise 0.

CString

Purpose

  • To expand on the string manipulation functions provided by Arduino String class.

  • To provide an alternative string class that does not use dynamic memory, as this can cause fragmentation of the limited memory available on Arduino MCUs..

  • Class CString uses CBuff and CFixedBuff as its internal array for storing characters if the string. These have a fixed maximum capacity. You can also use CDynamicBuff but doing so defeats the central purpose of CString in not using dynamic memory.

Dependancies: Arduino assert.h, IPAddress.h & Wire.h. LiquidCrystal_I2C.h for displaying debug output (optional & subject to define
                         in Constants.h).
                         Constants.h, CBuff.h/cpp, CString.h/cpp, debug.h/cpp, TextFile.h/cpp (optional & subject to a define at the top of debug.h)

CColor & Macros

Purpose: Inspired by Microsoft's RGB(...) macro, this library allows you to combine 8 bit red, green and blue colour
                values into a single 32 bit value. And asoextract individual 8 bit red, green and blue colour values from a single 32 bit value.
                The colour value encoding is designed to work with the Arduino library 'PololuAPA102' for RGB LED strips.
                Additional functionality includes obtaining opposite colours, random colours and stepping through the colours of the rainbow etc.

Macros & Functions

        #define RGB(nRed, nGreen, nBlue)

                 The macro takes 3 x 8 bit parameters - the red, green and blue values for the desired color. The blue value is stored in the first
                8 bits of a 32 bit value, the red value is stored in the second 8 bits and the green value is stored in the third 8 bits.
                The fourth 8 bits of the 32 bit value are unused.

                 Returns: the 32 bit coloir value.

        uint8_t GET_RED(uint32_t nColor)
       uint8_t GET_GREEN(uint32_t nColor)
       uint8_t GET_BLUE(uint32_t nColor)

                 These function return the 8 bit red, green and blue colour values from a combined 32 bit colour value.

                 Return: the 8 bit red, green or blue colour value.

      #define WHITE
      #define RED
      #define YELLOW
      #define GREEN
      #define CYAN
      #define BLUE
      #define MAGENTA
      #define BLACK

                 Return: 32 bit colour values for white, red, yellow, green, cyan, blue, magenta and black.

CColor

      uint32_t getRandomSecondaryColor()

                 Returns: a random color from blue, green, red, cyan, yellow or magenta.

      uint32_t getRandomColor()

                 Returns: a random color from a gradient of presets The presets are designed such that there is
                a visible difference in colour (when displayed on a RGB LED) between them.

      uint32_t getNextColor(const uint8_t nSteps = 1)

                 This function steps through the preset colour by the specified number of steps.

                 nSteps: The number of steps to move down the colour presets.
                Returns: one of the preset colours.

      uint16_t getColorCount()

                Returns: the number of preset colours.

      uint32_t getColor(const uint16_t nColorIndex)

                 Gets the preset colour from the specified index.

                 nColorIndex: index of the preset colour to fetch.

                 Returns: a preset colour value.

      uint32_t getColor(const char* cstrColorName)

                 Gets the appropriate colour value given the specified colour name.

                 cstrColorName: the name of the colour: red, green, blue, yellow, magenta, cyan, white, black.

                 Returns: a preset colour value.

      const char *getColorName(const uint32_t nColor)

                 Gets the appropriate colour name given the specified colour value.

                 nColor: the colour value.

                 Returns: the colour name - red, green, blue, yellow, magenta, cyan, white, black.

      bool isPrimaryColor(const uint32_t nColor)

                 nColor: the colour value to test.

      bool isSecondaryColor(const uint32_t nColor)

                 nColor: the colour value to test.

                 Returns: true if the color value correspnds to yellow, magenta, cyan or white.
