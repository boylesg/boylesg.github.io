Arduino C Libraries

These libraries are designed:

  •  To minimise memory usage - none of them have global objects like traditional Arduino libraries, and you must declare your own objects where approriate.

  • To make it quick and easy to put together new projects without having to re-code the same functionality over and over again.

  • The class interfaces are similar to, but expanded upon, those of Microsoft Foundation classes.

Well over a dozen of these libraries can be included in a given project, and still use well under 50% of a Mega's SDRAM memory space.

  • General Purpose

    • Array, Buffer, Color, Debug, Base64 Encode/Decode, I2C Scan, String, Date/Time, Map, Morse Code, Polytone, Power Control, Serial Data Manager, String Serial Source, Text File, Timer, UDP Email Client, UDP NTP Client, Voltage Divider.

  • ICs

    • ADC Manager for MCP3008, Manager for MCP41xxx Port Expander.
